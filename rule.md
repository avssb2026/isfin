# Правила проекта isfin (ипотека Мурабаха)

Постоянные ориентиры для изменений кода и контента. Не дублирует README — фиксирует соглашения, выросшие из реализации.

## Стек и структура

- **Next.js** (App Router), **TypeScript**, **Tailwind CSS** — публичные страницы и API в `src/app/`.
- **PostgreSQL** + **Prisma** — схема в `prisma/schema.prisma`, миграции в `prisma/migrations/`. После смены схемы: `npx prisma migrate dev` (или `deploy` на проде) и закоммитить SQL.
- **Auth.js (NextAuth v5)** — конфигурация в `src/auth.ts`, маршрут `src/app/api/auth/[...nextauth]/route.ts`. Закрытые разделы: префикс `/admin`, см. `middleware.ts`.

## Публичная часть и терминология

- Публичный текст держать в духе **Мурабаха** / рассрочка / цена / **платёж** / наценка банка. Не использовать в интерфейсе слова **«проценты»** и **«пени»** (требование ТЗ).
- **Ключевую ставку / расчётный годовой параметр не показывать клиенту** (используется только внутри расчёта).
- Калькулятор: общая логика в `src/lib/murabahaCalculator.ts`, UI — `src/components/MurabahaCalculator.tsx`. При изменении формулы — добавить/обновить тесты в `*.test.ts`.

## Чат с посетителем

- Виджет: `src/components/ChatWidget.tsx`, подключение через **`ConditionalChatWidget`** в `src/app/layout.tsx`, чтобы **не показывать чат на маршрутах `/admin`**.
- Пре-чат: имя и телефон → cookie-сессия → API `src/app/api/chat/`. Не ослаблять валидацию телефона и тела сообщений на сервере.

## Бэк-офис и CRM

- Список «Мои заявки» строится по **`assignedOperatorId === текущий пользователь`** (`src/app/api/admin/leads/route.ts`). Загрузка на клиенте — только при `useSession().status === "authenticated"`, запрос с **`credentials: "include"`**.
- Назначение ответственного и статус — `PATCH /api/admin/leads/[id]`, история — `ActivityLog`. Новые типы событий согласовывать с отображением в карточке лида.

## Безопасность

- Секреты только в переменных окружения; **`.env` не коммитить**. Публичные префиксы — `NEXT_PUBLIC_*`.
- Входы API проверять **Zod** (`src/lib/validations.ts`). Публичные эндпоинты — **rate limiting** (`src/lib/rate-limit.ts`). Пароли операторов — **argon2**, не хранить в открытом виде.
- **Edge middleware**: `middleware.ts` должен быть лёгким — не импортировать `src/auth.ts`, Prisma, `argon2` и другие тяжёлые зависимости. Проверку сессии делать через `next-auth/jwt`.
- CAPTCHA: подпись/проверка реализованы через **WebCrypto** (`crypto.subtle`) для совместимости с Edge; избегать Node `crypto` в коде, который может оказаться в Edge.

## Стиль кода

- Минимальный дифф: не рефакторить несвязанное. Следовать существующим именам и импортам. Комментарии — только где логика неочевидна.

## Локальная проверка по TDD

Цикл **красный → зелёный → рефакторинг** удобно гонять в watch-режиме Vitest (перезапуск при сохранении файлов).

1. **Красный:** добавить или изменить тест / ожидаемую пару вход–выход в `src/**/*.test.ts` так, чтобы требование ещё не выполнялось; убедиться, что прогон падает.
2. **Зелёный:** внести минимальные изменения в `src/lib/*` (или целевой модуль), пока `vitest` не покажет успех.
3. **Рефакторинг:** упростить код при зелёных тестах; не расширять объём задачи без нового теста.

Команды:

- `npm run test` — один прогон (как в CI), без watch.
- `npm run test:watch` или `npm run test:tdd` — интерактивный watch; остановка `q` или Ctrl+C.

Тесты без браузера: `environment: "node"` в `vitest.config.ts`. Логику держать в чистых функциях рядом с `*.test.ts`.
